# Fix: Invalid Server Actions Request Error in Booking Form

## Problem
The booking form submission is failing with "Invalid Server Actions request." error. This occurs when submitting the form in BookingForm.tsx, which uses a Server Action `createBooking` imported from `actions/booking.ts`.

The error is likely due to issues with Server Action handling in Next.js 16 with Turbopack, or improper action definition.

## Root Cause Analysis
- Server Actions in Next.js 16 with Turbopack can have compatibility issues.
- The `createBooking` function uses `bookingSchema.parse()`, which throws errors if validation fails, potentially causing the Server Action to fail.
- The form lacks an explicit `method="post"`, though Next.js should handle this automatically.
- Defining Server Actions in separate files may not be fully compatible in this setup.

## Solution
1. Move the `createBooking` Server Action directly into the `app/booking/page.tsx` file for better compatibility.
2. Add proper error handling using Zod's `safeParse` instead of `parse`.
3. Explicitly set `method="post"` on the form.
4. If issues persist, consider disabling Turbopack temporarily.

## Changes Required
### 1. Update `app/booking/page.tsx`
- Remove the import of `createBooking` from `@/actions/booking`.
- Add the Server Action definition directly in the file.
- Ensure the action is defined before the component.

### 2. Update `components/BookingForm.tsx`
- Add `method="post"` to the `<form>` element.

### 3. Optional: Disable Turbopack
If the above doesn't resolve the issue, disable Turbopack in `next.config.ts`:
```typescript
const nextConfig: NextConfig = {
  experimental: {
    turbo: {
      enabled: false,
    },
  },
};
```

## Implementation Steps
1. Move `createBooking` function to `page.tsx` with "use server"; directive.
2. Update form to include `method="post"`.
3. Test the form submission.
4. If error persists, disable Turbopack and test again.

- Ensure no "Invalid Server Actions request." error appears.
- Check browser network tab for successful POST request.
- Verify booking is inserted into Supabase database.
