# Plan: Admin Dashboard UI Refactoring

## Goal
Refactor the Admin Dashboard to be cleaner, more efficient, and professional. Remove the large hero banner, unify headers, and organize the layout with a sidebar approach.

## Changes

### 1. admin/page.tsx - Remove Hero Banner
- Delete the large `<section>` containing:
  - "Tableau de bord de l'enseignant" (p tag)
  - "Accès administrateur..." (h1)
  - Description paragraph
- Keep only the `<AdminPanel />` component wrapped in main

### 2. AdminLayout.tsx - Sidebar Layout
- Keep existing top bar (Logo left, Profile/Logout right)
- Add a vertical sidebar on the left with navigation:
  - "Tableau de Bord" icon (active state shows current page)
  - "Réservations" tab
  - "Ressources" tab
  - "Paramètres" (optional)
- Sidebar should be collapsible on mobile
- Main content area: use `bg-slate-900/50` background

### 3. BookingsTab.tsx - Compact Layout
- **Stats Cards**: Make them smaller with:
  - Smaller font for labels (text-xs instead of text-sm)
  - Less padding (p-3 or p-4 instead of p-5)
  - All 4 cards in single horizontal row
- **Action Bar**: Single horizontal row containing:
  - Search input (flex-1)
  - Status dropdown
  - "Actualiser" button
- **Page Header**: Replace "Gestion des réservations" title with breadcrumbs: "Tableau de bord / Réservations"

### 4. AdminPanel.tsx - Tab Integration
- Move the tab buttons (Réservations / Ressources) to the sidebar instead of inline buttons above content

### 5. Footer.tsx - Fix Constants Display
- Change `text-slate-400` to `text-slate-500` (reduced opacity)
- Make font smaller (text-xs)
- Optionally group into 3-column grid for better spacing

## Files to Modify
